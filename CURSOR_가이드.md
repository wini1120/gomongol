# Cursor 사용 가이드 (처음 사용자용)

## 1. Cursor란?
VS Code 기반 코드 에디터에 **AI 어시스턴트**가 붙어 있어, 채팅으로 질문하거나 코드를 선택해서 "이거 수정해줘", "이거 설명해줘"처럼 요청하면 바로 수정·설명을 해줍니다.

---

## 2. AI에게 요청할 때 이렇게 하세요

### ✅ 잘 하는 요청 예시
- **구체적으로**: "PostDetail.jsx 50번 줄 근처의 비밀번호 검증 로직에 5회 오류 시 잠금 기능 추가해줘"
- **파일/위치 명시**: "@PostDetail.jsx 이 파일에서 수정 버튼 클릭 시 로딩 스피너 넣어줘"
- **한 번에 하나씩**: "먼저 API 호출 부분만 리팩터링해줘. 그다음에 UI 수정할게"
- **목적을 말하기**: "모바일에서 게시글 카드가 너무 넓어서 2열 그리드로 바꿔줘"

### ❌ 피하면 좋은 요청
- "다 고쳐줘" (무엇을 어떻게 고칠지 모름)
- "빨리 해줘" (구체적 요구사항 없음)
- 여러 가지를 한 메시지에 다 넣기 (나눠서 요청하는 게 더 잘 됨)

### 💡 꿀팁
- **@파일이름** → 그 파일을 AI가 참고하게 할 수 있음 (예: `@src/App.jsx`)
- **@폴더/** → 폴더 전체 컨텍스트로 넣을 수 있음
- **코드 블록 선택 후** 질문하면 "이 부분만" 집중해서 답함

## 3. 자주 쓰는 단축키 (macOS 기준)

| 단축키 | 설명 |
|--------|------|
| **⌘ + L** | Cursor AI 채팅 패널 열기 (가장 많이 씀) |
| **⌘ + K** | 인라인 편집: 선택한 코드에 대해 바로 수정 요청 |
| **⌘ + I** | Composer 열기 (여러 파일 수정·신규 생성용) |
| **⌘ + Shift + L** | 새 채팅 시작 |
| **⌘ + /** | 선택 영역에 대해 질문 (코드 선택 후 사용) |
| **⌘ + .** | 현재 줄/선택 영역에 대한 빠른 수정 제안 |

- **채팅**: 보통 **⌘ + L** 로 열고, 요청 내용 입력 후 Enter.
- **코드 한 줄/블록 선택 → ⌘ + K**: "이 부분만 이렇게 바꿔줘" 할 때 유용.

## 4. 이 프로젝트(Gomongol) 요약

### 기술 스택
- **React 19** + **Vite** (빌드)
- **Tailwind CSS** (스타일)
- **Supabase** (DB·백엔드)
- **lucide-react** (아이콘), **html-to-image** (이미지 저장), **Vercel Analytics**

### 화면 구조 (App.jsx 기준)
1. **MainPage** – 메인: "일정 만들기", "지도 탐색", "동행찾기 게시판" 등 진입
2. **ItineraryBuilder** – 일정 빌더 (인원, 날짜, 지역/스팟 선택 후 게시글로 저장)
3. **Explorer** – 몽골 지도에서 지역별 스팟 탐색 (Supabase `master_spots` 사용)
4. **CommunityBoard** – 동행찾기 게시판 (검색·필터·페이지네이션, `posts` + `schedules`)
5. **PostDetail** – 게시글 상세·수정·삭제·비밀번호 확인·신고 등

### 데이터 (Supabase)
- **posts** – 게시글 (제목, 상태, 성별/연령 타겟, 인원, 설명, 채팅 링크 등)
- **schedules** – 일정 (출발일, 박수, 지역 ID, 스팟 ID 묶음)
- **master_regions** – 지역 마스터 (고비/중부/홉스굴 등)
- **master_spots** – 스팟 마스터 (지도 좌표 `x_coord`, `y_coord`, `region_id` 등)

### 참고할 점
- `explorer.jsx` 20번 줄: 지도 이미지가 `/mongol_map_clean.png` 인데, `public` 폴더에는 `mongol_map.png`, `mongol_map_A.png` 만 있음. 경로나 파일명 확인 필요할 수 있음.
- `mainpage.jsx` 에 "여행 후기 게시판" 메뉴는 있으나 라우팅/화면은 아직 없어 보임 (추가 개발 여지).

## 5. 앞으로 요청할 때 추천 문장

- "동행찾기 게시판에서 무한 스크롤로 바꿔줘"
- "PostDetail에서 수정 시 비밀번호 한 번만 맞으면 세션 동안 다시 안 물어보게 해줘"
- "Explorer 지도에 mongol_map_A.png 쓰도록 하고, 클릭 가능한 영역 추가해줘"
- "일정 빌더 2단계에서 선택한 지역별 스팟 개수 제한(예: 지역당 최대 5개) 걸어줘"

이렇게 **화면/파일 + 하고 싶은 일**을 같이 적어주면, 수정 포인트를 정확히 잡기 쉽습니다.
